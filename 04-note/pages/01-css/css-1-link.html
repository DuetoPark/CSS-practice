👻css 파일 정렬 순서

font - default(Reset css) - bootstrap - common - style 제일 나중에 선언된 css 파일이 우선순위 최상위 파일임. 우선순위: style > common > bootstrap > default > font

✏️css 파일 정렬 방법

  • import
  • link

🚗import보다 link를 권장하는 이유

import의 장점

-@impart가 css파일 내에서 모두 해결하기 때문에 더 깔끔해 보일 수 있다. 하지만 세상은 내 뜻대로 되지 않지 👅👅

import 단점

  1. 관리를 어렵게 한다.
  2. 마트료시카처럼 import안에 import 있고 안에 또 import 있기 무한 반복이라면 파악하기 힘들어진다.
  3. 파악만 불편해지면 다행! 스타일! 병합 발생! 불필요한 스타일 시트도 가져옴! 메이데이!
  4. 🔥로딩 속도를 늦춤🔥 link-병렬로 파일을 다운받음.(다운로드의 요청이 거의 동시에 이루어짐) import-우선 link된 파일 받고, 그 다음 안에 import된 스타일 시트를 받음.

✨스타일 시트를 무조건 여러개로 나누는 것이 좋을까?

가벼운 웹 페이지와 효율적인 관리를 위해서는 스타일 시트를 나누는 것이 좋다.
하지만 너무 잘게 자르면, 속도적인 측면에서 너무 느려짐.

이유)
웹 브라우저-파일 다운로드 받기 위해서 서버에 파일 요청.
서버에 요청하는 이 시간이 무시하지 못 할 정도로 커짐. 기다리던 사용자는 페이지를 나간다. 그럼 나도 회사를 나간다.. 타의적으로..
하나의 파일을 받는 것과 여러 개의 파일을 받는 것. 둘 중 더 빠른 것은 역시 하나의 파일을 받는 것이다.

정리)
-무의미할 정도의 시트 분할은 지양하자.
-정답은 없고 상황에 맞게 분할한다.
예를 들어, 본문 페이지에 방문 했더라도, 대부분 그 다음 다른 페이지에 접속하게 된다면, 오히려 그 두개의 페이지의 스타일 시트를 하나로 만드는 것이 나을 수 있다.

😎선택자 우선순위도 알아보자😎

구분 점수 더하는 방법
a 요소에 style attribute 쓰기(+1)
b id선택자 개수만큼 +1 (#a: 1, #a #b #c: 3)
c 클래스 선택자와 가상 클래스 선택자(:hover, :link 등등) 개수만큼 +1
d 요소 명으로 준 선택자와 가상요소 선택자 개수만큼 +1
